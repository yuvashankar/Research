228 /* Fills the edf_annotation_struct with the annotation n, returns 0 on success, otherwise -1 */
229 /* The string that describes the annotation/event is encoded in UTF-8 */
230 /* To obtain the number of annotations in a file, check edf_hdr_struct -> annotations_in_file. */
231 
232 /*
233 Notes:
234 
235 Annotationsignals
236 
237 EDFplus and BDFplus store the annotations in one or more signals (in order to be backwards compatibel with EDF and BDF).
238 The counting of the signals in the file starts at 0. Signals used for annotations are skipped by EDFlib.
239 This means that the annotationsignal(s) in the file are hided.
240 Use the function edf_get_annotation() to get the annotations.
241 
242 So, when a file contains 5 signals and the third signal is used to store the annotations, the library will
243 report that there are only 4 signals in the file.
244 The library will "map" the signalnumbers as follows: 0->0, 1->1, 2->3, 3->4.
245 This way you don't need to worry about which signals are annotationsignals. The library will do it for you.
246 
247 How the library stores time-values
248 
249 To avoid rounding errors, the library stores some timevalues in variables of type long long int.
250 In order not to loose the subsecond precision, all timevalues have been multiplied by 10000000.
251 This will limit the timeresolution to 100 nanoSeconds. To calculate the amount of seconds, divide
252 the timevalue by 10000000 or use the macro EDFLIB_TIME_DIMENSION which is declared in edflib.h.
253 The following variables do use this when you open a file in read mode: "file_duration", "starttime_subsecond" and "onset".
254 */
