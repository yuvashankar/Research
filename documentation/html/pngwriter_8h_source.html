140  /* General Notes
141  * It is important to remember that all functions that accept an argument of type "const char *" will also
142  * accept "char *", this is done so you can have a changing filename (to make many PNG images in series
143  * with a different name, for example), and to allow you to use string type objects which can be easily
144  * turned into const char * (if theString is an object of type string, then it can be used as a const char *
145  * by saying theString.c_str()).
146  * It is also important to remember that whenever a function has a colour coeffiecient as its argument,
147  * that argument can be either an int from 0 to 65535 or a double from 0.0 to 1.0.
148  * It is important to make sure that you are calling the function with the type that you want.
149  * Remember that 1 is an int, while 1.0 is a double, and will thus determine what version of the function
150  * will be used. Similarly, do not make the mistake of calling for example plot(x, y, 0.0, 0.0, 65535),
151  * because
152  * there is no plot(int, int, double, double, int).
153  * Also, please note that plot() and read() (and the functions that use them internally)
154  * are protected against entering, for example, a colour coefficient that is over 65535
155  * or over 1.0. Similarly, they are protected against negative coefficients. read() will return 0
156  * when called outside the image range. This is actually useful as zero-padding should you need it.
157  * */

198  /* Plot
199  * With this function a pixel at coordinates (x, y) can be set to the desired colour.
200  * The pixels are numbered starting from (1, 1) and go to (width, height).
201  * As with most functions in PNGwriter, it has been overloaded to accept either int arguments
202  * for the colour coefficients, or those of type double. If they are of type int,
203  * they go from 0 to 65535. If they are of type double, they go from 0.0 to 1.0.
204  * Tip: To plot using red, then specify plot(x, y, 1.0, 0.0, 0.0). To make pink,
205  * just add a constant value to all three coefficients, like this:
206  * plot(x, y, 1.0, 0.4, 0.4).
207  * Tip: If nothing is being plotted to your PNG file, make sure that you remember
208  * to close() the instance before your program is finished, and that the x and y position
209  * is actually within the bounds of your image. If either is not, then PNGwriter will
210  * not complain-- it is up to you to check for this!
211  * Tip: If you try to plot with a colour coefficient out of range, a maximum or minimum
212  * coefficient will be assumed, according to the given coefficient. For example, attempting
213  * to plot plot(x, y, 1.0,-0.2,3.7) will set the green coefficient to 0 and the red coefficient
214  * to 1.0.

333  * that PNG variants are supported. Very generally speaking, most PNG files can now be read (as of version 0.5.4),

627  /* Laplacian
628  * This function applies a discrete laplacian to the image, multiplied by a constant factor.
629  * The kernel used in this case is:
630  * 1.0 1.0 1.0
631  * 1.0 -8.0 1.0
632  * 1.0 1.0 1.0
633  * Basically, this works as an edge detector. The current pixel is assigned the sum of all neighbouring
634  * pixels, multiplied by the corresponding kernel element. For example, imagine a pixel and its 8 neighbours:
635  * 1.0 1.0 0.0 0.0
636  * 1.0 ->1.0<- 0.0 0.0
637  * 1.0 1.0 0.0 0.0
638  * This represents a border between white and black, black is on the right. Applying the laplacian to
639  * the pixel specified above pixel gives:
640  * 1.0*1.0 + 1.0*1.0 + 0.0*1.0 +
641  * 1.0*1.0 + 1.0*-8.0 + 0.0*1.0 +
642  * 1.0*1.0 + 1.0*1.0 + 0.0*1.0 = -3.0
643  * Applying this to the pixel to the right of the pixel considered previously, we get a sum of 3.0.
644  * That is, after passing over an edge, we get a high value for the pixel adjacent to the edge. Since
645  * PNGwriter limits the colour components if they are off-scale, and the result of the laplacian
646  * may be negative, a scale factor and an offset value are included. This might be useful for
647  * keeping things within range or for bringing out more detail in the edge detection. The
648  * final pixel value will be given by:
649  * final value = laplacian(original pixel)*k + offset
650  * Tip: Try a value of 1.0 for k to start with, and then experiment with other values.
651  * */
652  void laplacian(double k, double offset);
